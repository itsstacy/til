## 프레임워크/언어

### React/javascript

- 리액트는 가장 많이 사용하는 SPA 도구이며 레퍼런스가 다양하다.

## 라이브러리

### redux

- 제일 기본적인 상태관리 라이브러리. 가장 많이 사용하기 때문에 레퍼런스가 많다.

### redux toolkit

- redux-thunk, immer 등 추가 패키지들이 다 포함되어 있어 따로 설치하지 않아도 되고 redux의 boillerplate를 짧게 줄일 수 있다.

### Axios

- 브라우저, Node.js를 위한 Promise API를 활용하는 HTTP 비동기 통신 라이브러리.

### Wavesurfer

- 맞춤형 오디오 파형 시각화 라이브러리. (영어..천국..저는..또르르..)
- 사용자가 업로드한 오디오 파일을 시각적으로 보여주기 위해 사용.

### Stomp, Sock

- 소켓 통신 기반으로 실시간 채팅, 실시간 알림 등을 구현할 수 있다.
- stomp의 경우 java에 종속적이라 react-spring 환경에서 많이 사용하고 있는 것으로 보인다.
- [socket.io](http://socket.io) 보다 가볍고 subscribe 방식이라 더 안정적인 구동이 기대된다.

### OpenVidu

- Kurento 기반의 중개 서버를 에플리케이션에 추가할 수 있다.
- 세션 별 실시간 영상 스트리밍이 가능
- 웹소켓이 아닌 브라우저끼리 연결시키는 WebRTC 방법을 이용해 데이터 스트림이 가능
- 영상 스트리밍 뿐만 아니라 화면 공유, 메시지 브로드 캐스팅과 같이 화상 회의 애플리케이션에서 필요한 대부분의 기능들을 소스코드로 제공
- Openvidu Browser 과 Openvidu Server 두 파트로 나뉜다.

### Styled-components

- Javascript 파일 내에서 CSS를 사용할 수 있게 해주는 대표적인 CSS-in-JS 라이브러리로 React 프레임워크를 주요 대상으로 한 라이브러리.
- css 파일을 밖에 두지 않고, 컴포넌트 내부에 넣기 때문에, css가 전역으로 중첩되지 않도록 만들어주는 장점이 있다.
- props로 상태값을 내려주어 상태값에 따른 css 변경이 가능하다.

### React-slick, React-carousel

- React-slick를 사용해 캐로셀(슬라이드)을 쉽게 구현할 수 있음 (slick-carousel 모듈도 함께 설치해야 함)
- React-slick은 캐로셀을 구현하는 방법 중 가장 대중적이고 자료가 많음

### Moment

- JavaScript에서 사용되는 날짜와 시간 관련 라이브러리 중 가장 많이 사용
- Moment.js를 사용하면 별도의 함수를 만들 필요없이 내장 함수를 이용해 날짜와 시간을 나타낼 수 있음

### Dotenv

- 환경 변수를 파일에 저장할 수 있도록 해줌
- 깃허브에 올리기 전 민감한 정보 (서버 주소, 엑세스 키 등)를 숨겨두기 위해 사용
- .env 파일을 만들어서 환경변수를 설정하고 process.env를 사용해서 불러올 수 있다.
- 주의할 점: 환경변수는 적용이 되려면 선언한 다음 프로젝트를 껐다가 켜야 한다.

### React-colorful

- 사용자가 원하는 색상을 선택하는 방식의 도구를 적용하고 싶어 사용한 색상 선택 라이브러리

### React-icons

- 다른 아이콘 라이브러리보다 적용하기 쉽고 간단하게 아이콘을 삽입할 수 있기 때문에 사용

## 협업툴

### Github

- 버전 관리와 협업을 위한 웹 호스팅 플랫폼으로 프로젝트를 쉽게 진행할 수 있도록 돕는 역할
- 원하는 시점으로 복원이 가능하고 수정한 내용에 대한 문서화도 용이함
- 여러 사람이 하나의 소스를 작업해도 충돌 방지 및 해결이 쉬움, 
누가 어떤 작업을 진행했는지 확인이 가능
- 다수의 오픈 소스 저장소를 보유하고 있음

### Notion

- 간단한 메모부터 프로젝트 관리, 엑셀 및 간단한 코딩 작업 그리고 자료 보관까지 하나의 프로그램에서 구현 가능
- 다양한 템플릿을 제공하며, 노트 관리가 유연하고 체계적인 편
- 개인화가 가능함

## CI/CD

### Github Action

- CI/CD용 툴
- 깃허브에서 코드를 같이 관리하면서 배포시점 까지 설정할 수 있어서 편하다.
- 깃허브에서 develop과 master 두 가지 브랜치로 관리하는데, develop 브랜치를 default로 설정하고, 실제로 배포되는 브랜치는 master로 설정했다.
